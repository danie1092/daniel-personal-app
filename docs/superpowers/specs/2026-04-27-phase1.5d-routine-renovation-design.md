## 배경

루틴 페이지는 사용자가 매일 체크 + 월별 추세를 보고 싶어 하는 곳. 사용자가 첫 화면에서 보여준 **해빗 트래커 그리드 + Done/Progress 행 + 추세 그래프** 컨셉을 토스 스타일로 적용한다.

기존 페이지에 다마고치 잔재(EXP/Level/pokemon_id 시스템)가 있으나 Phase 1에서 진입점이 모두 제거됐으므로 코드에서 deadcode로 두지 말고 깔끔히 제거. DB 컬럼은 Phase 2+에서 drop.

## 결정 사항 (자동)

### 디자인
- **3탭 유지**: 체크 / 그래프 / 설정 — 가계부와 동일 패턴
- **기본 탭 = 체크** (사용자 의도 = 매일 체크가 메인)
- **헤더**: "루틴" + 오늘 날짜 + 큰 진행률 도넛 (오늘 N/M, X%)
- **체크 탭**: 항목 리스트 (이모지 + 이름 + 체크박스). 체크박스 색은 `success #22C55E` (Phase 1과 통일). 체크 사운드 유지(`acSound`).
- **그래프 탭**:
  - **월 선택 pill** (가계부와 동일 패턴)
  - **해빗 트래커 그리드**: 행 = 항목, 열 = 일(1~31). 가로 스크롤. 셀 = 체크/미체크. 우측에 Total(체크된 일수)
  - **Done 행**: 당일 체크된 항목 수 (그리드 하단)
  - **Progress 행**: 당일 % (그리드 하단)
  - **추세 라인 차트**: 일별 Progress % 30일 (smooth curve, success 컬러)
  - **연속 일수 (Streak)**: 항목별 현재 연속 일수, 큰 순 정렬, 별도 카드
- **설정 탭**: 항목 add/edit/delete/reorder. 토스 스타일 sheet로 add/edit. 순서는 위/아래 화살표

### Server Actions
```ts
toggleRoutineCheck(itemId, date, checked): {ok, checked} | {ok:false, error}
createRoutineItem({name, emoji}): {ok, id} | error
updateRoutineItem(id, {name, emoji}): {ok} | error
deleteRoutineItem(id): {ok} | error
moveRoutineItem(id, direction: "up"|"down"): {ok} | error
```

모든 Action: `requireSession` + 입력 검증 (name ≤ 100, emoji ≤ 16) + revalidatePath(/routine, /home).

### 그래프 탭 데이터 헬퍼

```ts
// src/lib/routine/graphData.ts
export type DayCell = { date: string; checked: boolean };
export type ItemRow = { item: RoutineItem; cells: DayCell[]; total: number };
export type DayTotal = { date: string; done: number; total: number; pct: number };
export type ItemStreak = { item: RoutineItem; currentStreak: number };

export type MonthGraphData = {
  yearMonth: string;
  items: RoutineItem[];
  rows: ItemRow[];           // 항목 × 일자별 cells + Total
  dayTotals: DayTotal[];     // 각 일자의 Done/Total/Progress
  streaks: ItemStreak[];     // 항목별 현재 연속 일수 (큰 순)
};

export async function getMonthGraphData(yearMonth: string): Promise<MonthGraphData>;
```

연속 일수 계산: 오늘부터 거꾸로 연속된 체크 일수. 미래 날짜는 무시. 일부 항목 미체크면 streak 끊김.

## 위험 / 트레이드오프
- **해빗 그리드 가로 스크롤** — 30일 × 10항목 = 300 셀. iPhone에서 셀이 작아질 수 있음. 셀 크기 24px 정도, 한 화면에 7~8일 노출, 가로 스크롤로 나머지.
- **그래프 탭 데이터 양** — 한 번 fetch에 routine_items + routine_checks 한 달분. 보통 10항목 × 30일 = 300 row. 여유.
- **사운드 무음 처리** — iOS Safari 자동재생 정책: 사용자 인터랙션 후에만 사운드. 체크 클릭이 인터랙션이라 OK.
